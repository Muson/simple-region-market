Standard Templates

You can define templates in your 'templates.yml'. That means you can change names, create new types and set time or pricing limits.

[SELL]

For selling regions.
The buyer will be owner if nothing else is defined in the templates file.

[LET]

For renting out regions.
The renter will be member and the rent will be automatically extended, if the renter has got enough money. If the renter hasn't got enough money, the region will be released directly.

[HOTEL]

For renting out regions.
The renter will be member and needs to reclick the sign (and therefor repay), if he wants to extend his time on the region. If the time runs out, the region will be released directly.

Permissions

Other permissions than simpleregionmarket.admin can be set for all templates you define. That means (replace name with the name of the template):

  • simpleregionmarket.name.take - Permission to let a player buy/rent a region
  • simpleregionmarket.name.offer - Permission to let a player sell/let a region
  • simpleregionmarket.name.addmember - Permission to let a player add/remove members to/from his regions
  • simpleregionmarket.name.addowner - Permission to let a player add/remove owners to/from his regions

Limits

You can set limits when you create the limits.yml manually (example below; use the YAML format; -1 means often infinite; If you dont know how: Let it).

limits.yml (name means the name of the template; world means the world; etc.)

  • global.global - Global limit
  • name.global - Global template limit
  • global.world.world - Global world limit
  • name.world.world - Template world limit
  • global.parentregion.parentregion - Global limit by parentregion (NOT priority)
  • name.parentregion.parentregion - Template limit by parentregion
